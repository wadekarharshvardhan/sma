UML 1: Library Management System — Use Case Diagram

1. Theory: What Is a Use Case Diagram?
A Use Case Diagram shows:

Who interacts with the system (Actors)
What they can do (Use Cases)
How actors and use cases are related

It represents the functional requirements of a system from a user's perspective.

7. Viva Questions
Q1: What is an actor?

An entity (person or system) that interacts with the system from outside.

Q2: What is the difference between include and extend?

<<include>> is mandatory — always happens. <<extend>> is optional — happens only under certain conditions.

Q3: Why is "Verify Member" an include of "Issue Book"?

Because verification MUST happen every time a book is issued — it's mandatory.

Q4: Why is "Calculate Fine" an extend of "Return Book"?

Fine is calculated ONLY IF the book is returned late — it's optional/conditional.

Q5: What is a system boundary?

A rectangle that shows what functionality is inside the system vs. outside.
