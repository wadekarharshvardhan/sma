🧪 EXPERIMENT 3: Prim's Minimum Spanning Tree

1. What Problem Are We Solving?
Given a weighted undirected graph, we want to connect all nodes using edges with the minimum total weight, without forming any cycle. This is called a Minimum Spanning Tree (MST).
Real example: Connect all cities with roads using minimum total road length.

2. Why Greedy Approach?
Prim's is a greedy algorithm — at every step, it picks the cheapest available edge that connects the already-built tree to a new node. This locally optimal choice leads to a globally optimal MST.

3. Real-World Applications

Network design (minimum cable to connect all computers)
Road/pipeline construction
Cluster analysis in machine learning
Circuit board design


4. Detailed Theory
Spanning Tree:

A tree that includes all vertices of the graph
Has exactly V-1 edges (V = number of vertices)
Has no cycles

Minimum Spanning Tree:

A spanning tree with the minimum sum of edge weights

Prim's Algorithm:

Start from any node
Maintain a set of visited nodes
At each step, pick the minimum weight edge that connects visited to unvisited
Add that edge and new node to the tree
Repeat until all nodes are in the tree


5. Step-by-Step Logic
1. Start with any node (say node 0)
2. Mark it as visited
3. Add all its edges to a priority queue
4. While priority queue is not empty:
   a. Pick edge with minimum weight
   b. If destination node already visited → skip
   c. Mark destination as visited
   d. Add this edge to MST
   e. Add all edges of new node to priority queue
5. Print MST edges and total weight

6. Beginner-Friendly Code with Comments
pythonimport heapq  # For priority queue

# ============================================================
# PRIM'S ALGORITHM
# ============================================================
def prims_mst(graph, start=0):
    """
    graph: adjacency list {node: [(weight, neighbor), ...]}
    start: starting node (default 0)
    """
    n = len(graph)  # Number of nodes

    visited = set()    # Nodes already in MST
    visited.add(start) # Start with the first node

    # Priority queue: (weight, from_node, to_node)
    # We start by adding all edges from the start node
    min_heap = []
    for weight, neighbor in graph[start]:
        heapq.heappush(min_heap, (weight, start, neighbor))

    mst_edges = []   # List to store MST edges
    total_weight = 0 # Total weight of MST

    print("Prim's MST:")

    # Keep picking minimum edges until all nodes visited
    while min_heap and len(visited) < n:
        # Get the minimum weight edge
        weight, from_node, to_node = heapq.heappop(min_heap)

        # If to_node already in MST → skip (would create cycle)
        if to_node in visited:
            continue

        # Add to_node to MST
        visited.add(to_node)
        mst_edges.append((from_node, to_node, weight))
        total_weight += weight

        print(f"  Edge: {from_node} --{weight}--> {to_node}")

        # Add all edges from newly added node to the heap
        for next_weight, next_neighbor in graph[to_node]:
            if next_neighbor not in visited:
                heapq.heappush(min_heap, (next_weight, to_node, next_neighbor))

    print(f"Total MST Weight: {total_weight}")
    return mst_edges, total_weight

10. Viva Questions
Q1: What is a Minimum Spanning Tree?

A tree connecting all vertices with minimum total edge weight and no cycles.

Q2: Why is Prim's called greedy?

It always picks the minimum weight edge at each step without looking ahead.

Q3: How many edges does an MST have?

V-1 edges where V is the number of vertices.

Q4: What is the difference between Prim's and Kruskal's?

Prim's builds tree from a node, adding nearest neighbor. Kruskal's sorts all edges and adds them if no cycle is formed.

Q5: What happens if graph is disconnected?

MST cannot be formed — we get a Minimum Spanning Forest instead.

Mod: Kruskal's instead of Prim's
pythondef kruskals_mst(edges, n):
    # edges = [(weight, u, v), ...]
    edges.sort()  # Sort by weight
    parent = list(range(n))  # Union-Find parent array

    def find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]

    def union(x, y):
        parent[find(x)] = find(y)

    mst = []
    for weight, u, v in edges:
        if find(u) != find(v):  # No cycle
            union(u, v)
            mst.append((u, v, weight))
    return mst

🧪 EXPERIMENT 4: Constraint Satisfaction — N-Queens with Backtracking

1. What Problem Are We Solving?
Place N queens on an N×N chessboard such that no two queens attack each other.
Queens attack in: same row, same column, same diagonal.

2. Why Backtracking?
We try placing queens one by one. If a placement leads to a conflict → we backtrack (undo) and try a different position. This avoids exploring impossible paths.

3. Real-World Applications

Scheduling problems (assign tasks without conflicts)
Map coloring
Sudoku solving
Circuit design
Timetable creation


4. Detailed Theory
Constraint Satisfaction Problem (CSP):

Variables: Position of each queen (one per row)
Domain: Column positions (0 to N-1)
Constraints: No two queens in same column or diagonal

Backtracking:

A systematic trial and error method
Try a value → check constraints → if valid continue → if invalid try next value
If no value works → go back to previous variable

Branch and Bound:

Extension of backtracking
Prunes branches that cannot possibly lead to a solution
Uses a bound to decide early if a path is hopeless


5. Step-by-Step Logic
1. Start from row 0
2. Try placing queen in each column of current row
3. Check if placement is safe (no conflict with previously placed queens)
4. If safe → place queen → move to next row
5. If not safe → try next column
6. If all columns tried and none safe → backtrack to previous row
7. If all N queens placed → solution found!

6. Beginner-Friendly Code with Comments
python# ============================================================
# N-QUEENS PROBLEM — BACKTRACKING SOLUTION
# ============================================================

def is_safe(board, row, col, n):
    """
    Check if placing a queen at (row, col) is safe.
    We only check previously placed queens (rows 0 to row-1).
    """

    # Check same column — scan all rows above
    for i in range(row):
        if board[i] == col:  # Another queen in same column
            return False

    # Check upper-left diagonal
    # Move diagonally up-left: row decreases, col decreases
    i, j = row - 1, col - 1
    while i >= 0 and j >= 0:
        if board[i] == j:  # Queen found on this diagonal
            return False
        i -= 1
        j -= 1

    # Check upper-right diagonal
    # Move diagonally up-right: row decreases, col increases
    i, j = row - 1, col + 1
    while i >= 0 and j < n:
        if board[i] == j:  # Queen found on this diagonal
            return False
        i -= 1
        j += 1

    return True  # Safe to place queen here


def solve_nqueens(board, row, n, solutions):
    """
    Recursively place queens row by row.
    board[i] = column where queen is placed in row i
    """

    # BASE CASE: All N queens placed successfully
    if row == n:
        solutions.append(board[:])  # Save a copy of this solution
        return

    # Try placing queen in each column of current row
    for col in range(n):
        if is_safe(board, row, col, n):
            board[row] = col          # Place queen at (row, col)
            solve_nqueens(board, row + 1, n, solutions)  # Recurse to next row
            board[row] = -1           # BACKTRACK: Remove queen (try next column)


def print_board(board, n):
    """Print the chessboard with Q for queen and . for empty"""
    for row in range(n):
        line = ""
        for col in range(n):
            if board[row] == col:
                line += "Q "  # Queen
            else:
                line += ". "  # Empty
        print(line)
    print()

if __name__ == "__main__":

10. Viva Questions
Q1: What is backtracking?

A technique that tries placing values, and if constraints are violated, undoes (backtracks) and tries next option.

Q2: What are the constraints in N-Queens?

No two queens can share the same row, column, or diagonal.

Q3: Why don't we check rows?

We place exactly one queen per row by design — row conflicts can't happen.

Q4: How many solutions exist for 8-Queens?

92 solutions.

Q5: What is the difference between backtracking and brute force?

Backtracking prunes invalid branches early; brute force tries all possibilities regardless.

Q6: What is a CSP?

Constraint Satisfaction Problem — find values for variables satisfying all constraints.

Q7: What is Branch and Bound?

An extension of backtracking that uses a cost bound to prune branches that cannot improve the current best solution.

pythondef graph_coloring(graph, colors, node, num_colors, color_map):
    if node == len(graph):
        return True  # All nodes colored

    for color in range(1, num_colors + 1):
        if is_color_safe(graph, color_map, node, color):
            color_map[node] = color
            if graph_coloring(graph, colors, node + 1, num_colors, color_map):
                return True
            color_map[node] = 0  # Backtrack

    return False

def is_color_safe(graph, color_map, node, color):
    for neighbor in graph[node]:
        if color_map[neighbor] == color:
            return False
    return True

12. Memory Tricks
Backtracking = "Try, Fail, Go Back, Try Again"
Like solving a maze: hit a dead end → step back → try new path

N-Queens checks:
1. Same column: board[i] == col
2. Left diagonal: board[i] == col - (row - i)
3. Right diagonal: board[i] == col + (row - i)

Or simply: check col, check both diagonals using while loops
