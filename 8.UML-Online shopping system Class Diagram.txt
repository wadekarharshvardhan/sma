UML 2: Online Shopping System — Class Diagram

1. Theory: What Is a Class Diagram?
A Class Diagram shows:

Classes (blueprints for objects)
Attributes (data/properties)
Methods (operations/functions)
Relationships between classes

It represents the static structure of the system.

5. Viva Questions
Q1: What is the difference between aggregation and composition?

Aggregation: weak relationship — part can exist independently (Cart and Product). Composition: strong — part cannot exist without whole (Order and OrderItem, if order deleted, items deleted too).

Q2: What is an interface in class diagram?

A contract that classes must implement. Shown with <<interface>> stereotype.

Q3: What is multiplicity?

Numbers on relationship lines showing how many objects relate. E.g., 1..* means one to many.

Q4: What is the difference between association and dependency?

Association is a structural relationship (permanent). Dependency is temporary (one class uses another briefly).

1. Theory: What Is a Sequence Diagram?
A Sequence Diagram shows how objects interact in time order — the sequence of messages passed between them to complete a use case.

4. Viva Questions
Q1: What does a sequence diagram show?

The order of messages/interactions between objects over time for a specific scenario.

Q2: What is a lifeline?

A dashed vertical line below an object showing its existence throughout the interaction.

Q3: What is an activation bar?

A thin rectangle on a lifeline showing when an object is actively processing.

Q4: What is the difference between sequence and collaboration diagram?

Sequence shows time order of messages. Collaboration shows structural relationships and messages together.
